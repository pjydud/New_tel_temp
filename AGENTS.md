# New_tel_temp PM & Agent 운영 가이드

이 저장소는 사용자가 요구사항을 말하면 PM이 정리하고, 기능별 Agent가 역할을 나누어 구현하는 방식으로 운영한다.

## PM 역할

PM은 전체 방향을 관리한다.

- 사용자 요구사항 정리
- 우선순위 결정
- 기능 범위 확정
- GitHub 변경사항 검토
- 배포 후 테스트 항목 정리
- 문제 발생 시 원인 분리

## Agent 구조

### 1. News Agent

역할:

- 구글 뉴스 RSS 수집
- 인물별 뉴스 검색
- 화성시 정책·민원 뉴스 검색
- 최근 7일 기사 필터링
- 중복 기사 제거

주요 파일:

- `api/send-news.js`

### 2. Political Agent

역할:

- 권칠승 국회의원
- 경기도의원
- 화성시의원
- 정명근 화성시장 관련 뉴스 확인
- 이름이 제목에 포함된 기사 우선 수집

### 3. Hwaseong Policy Agent

역할:

- 화성시 정책
- 화성시의회 조례·예산
- 교통·주차·도시계획·개발·GTX·반도체 산업단지 관련 뉴스 수집
- 단순 행사·봉사·협약 뉴스 제외

### 4. Stock Agent

역할:

- 네이버 금융 기준 코스피·코스닥 거래상위 종목 수집
- ETF, ETN, 레버리지, 인버스, 선물, 스팩 등 제외 시도
- 투자 참고용 문구 포함

### 5. Telegram Agent

역할:

- 텔레그램 메시지 전송
- HTML 링크 형식 적용
- 긴 URL 숨김
- 기사 제목 클릭 시 원문 이동
- 여러 채팅방 동시 발송

환경변수:

```env
TELEGRAM_BOT_TOKEN=봇토큰
TELEGRAM_CHAT_ID=기본 개인방 또는 기본 그룹방 ID
TELEGRAM_CHAT_IDS=여러 방에 보낼 경우 콤마로 구분
```

예시:

```env
TELEGRAM_CHAT_IDS=5287488983,-5595644220
```

`TELEGRAM_CHAT_IDS`가 있으면 이 값을 우선 사용한다. 없으면 `TELEGRAM_CHAT_ID`와 기본 테스트방을 함께 사용한다.

### 6. Monitoring Agent

역할:

- 정해진 시간에 API 호출
- Vercel Cron 또는 외부 스케줄러와 연동
- 실패 시 에러 메시지 확인

## 현재 메시지 발송 흐름

```text
사용자 요청
↓
PM 요구사항 정리
↓
News / Political / Hwaseong / Stock Agent 데이터 수집
↓
Telegram Agent 메시지 구성 및 발송
↓
개인방 + 그룹방 동시 전송
```

## 변경 작업 원칙

1. 기존 기능을 가능한 한 깨지 않는다.
2. 작은 단위로 수정한다.
3. 메시지 전송 관련 변경은 항상 테스트방부터 확인한다.
4. 봇 토큰은 GitHub에 직접 저장하지 않는다.
5. 링크는 긴 URL을 노출하지 않고 제목 클릭 방식으로 보낸다.
6. 여러 방 전송 시 실패한 방이 있으면 에러 내용을 확인한다.
